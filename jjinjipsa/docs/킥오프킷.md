## [1] CLAUDE.md (프로젝트 루트에 저장)

```markdown
# 찐집사 (Jjinjipsa)

한국 고양이 집사를 위한 건강 케어 웹앱. "고양이 건강 정보의 화해" —
출처 있는 건강 정보와 각 고양이의 기록을 결합한 서비스.

## 먼저 읽을 문서
- docs/PRD_원페이저.md — 제품 비전, 브랜드 메시지, 수익화
- docs/기능명세서.md — 전체 기능 명세 F-01..F-16, DB 스키마, 릴리즈 계획
- docs/찐집사_목업.html — 동작하는 인터랙티브 목업. 이것이 비주얼 스펙이다:
  디자인 토큰, 아바타 생성기(catSVG), 나이 환산 로직, 화면 플로우 전부 포함.
  새로 만들지 말고 여기서 로직과 스타일을 이식할 것.
- docs/케어카드_샘플_10장.md + docs/입양준비_가이드카드.md — 시드 콘텐츠 (16장)
- docs/챗봇_시스템프롬프트.md — 챗봇 시스템 프롬프트 (v1.1용, MVP 아님)

## 스코프 규율 (중요)
- MVP = F-01..F-05 + F-07 + F-15. MVP에는 Supabase, 로그인, 챗봇을 넣지 않는다.
- MVP의 모든 저장은 localStorage. 저장 계층은 엔티티별 get/set을 가진 얇은
  어댑터로 설계해서, v1.1에서 UI 코드를 건드리지 않고 Supabase로 교체할 수
  있게 한다.
- 케어 카드는 docs 콘텐츠에서 변환한 정적 JSON. CMS는 아직 없다.

## 스택 & 컨벤션
- Next.js 14+ App Router, TypeScript, Tailwind CSS. 배포는 Vercel.
- 모바일 퍼스트, 데스크톱에선 최대 420px 중앙 셸 (목업의 .phone 프레임처럼).
- 디자인 토큰은 목업 기준: pine #1E5C4F, pine-deep #123B32,
  pine-soft #DCEAE4, churu #F5C956, churu-soft #FBEFD2, salmon #F08A72,
  bg #F2F6F3, ink #1C2B27. 폰트: Pretendard Variable (CDN).
- UI 카피는 한국어. 목업에 있는 문구는 그대로 재사용한다.
- 아바타 시스템(털색 × 무늬 × 눈 SVG 생성기)은 핵심 브랜드 자산이다.
  목업의 catSVG를 components/CatAvatar.tsx로 이식하고 타입 있는 props를 준다.
- 모든 건강 정보 카드는 source_name(출처)을 반드시 렌더링한다. 페이지 하단
  고정 면책 문구: "이 정보는 참고용이며, 정확한 진단은 수의사 상담이 필요합니다."
- 귀여운 마이크로 인터랙션은 장식이 아니라 제품 요구사항이다: 꾹꾹이 로더,
  체크리스트 완료 발도장, 말풍선 탭 시 귀 쫑긋.
- prefers-reduced-motion을 존중한다.

## 데이터 모델 (MVP, localStorage 키)
- jjinjipsa:cats — 묘 프로필 배열 (기능명세서 §5 `cats` 참고)
- jjinjipsa:traits:{catId} — 오늘의 체크에서 쌓이는 습관 프로필 (F-15)
- jjinjipsa:symptoms:{catId} — 증상 로그 (F-05)
- jjinjipsa:reads:{catId} — 카드 읽음/저장 상태

## 기능별 완료 기준
docs/기능명세서.md의 해당 F-번호에 적힌 항목들과 일치할 것.
마일스톤 완료 판단 전에 반드시 `npm run build` 통과 확인.
```

## [2] 첫 세션 프롬프트 (M0 — 뼈대)

```text
CLAUDE.md와 docs 폴더를 먼저 읽어줘. 특히 docs/찐집사_목업.html.

마일스톤 M0 — 프로젝트 뼈대:

1. 현재 디렉토리에 Next.js 14 App Router + TypeScript + Tailwind 프로젝트를
   스캐폴드해줘. CLAUDE.md의 디자인 토큰을 Tailwind 테마 컬러로 설정하고
   Pretendard Variable을 로드해.
2. 모바일 앱 셸을 만들어줘: 데스크톱에선 420px 중앙 프레임, 모바일에선
   전체 화면. 목업의 .phone 컨테이너와 동일하게.
3. 목업의 catSVG 함수를 components/CatAvatar.tsx로 이식해줘:
   타입 있는 props { fur, pattern, eyes, mode, size },
   털색 6종 × 무늬 6종 × 눈 4종. 삼색/젖소의 흰 베이스 강제 규칙과
   포인트 무늬의 어두운 귀 규칙까지 포함해서. 그리고 모든 조합을
   그리드로 렌더링하는 /dev/avatar 플레이그라운드 페이지를 만들어서
   내가 눈으로 QA할 수 있게 해줘.
4. localStorage 어댑터(lib/storage.ts)를 만들어줘. CLAUDE.md의 4개 키에
   대한 타입 있는 get/set. 나중에 같은 인터페이스 뒤에서 Supabase
   어댑터로 교체할 수 있는 구조로.
5. 시드 데이터: docs/케어카드_샘플_10장.md와 docs/입양준비_가이드카드.md의
   카드 16장을 기능명세서 §5의 care_cards 스키마(title, body, source_name,
   tags_stage, tags_breed, tags_condition, severity)에 맞춰
   data/cards.json으로 변환해줘.
6. npm run build 통과를 확인하고, Vercel 배포 명령어 목록을 알려줘.

온보딩 화면은 아직 만들지 마 — 그건 M1이야. 이번 마일스톤은 뼈대,
아바타 컴포넌트, 저장 어댑터, 시드 데이터까지만.
```

## [3] 이후 세션 프롬프트 (한글)

**M1 — 온보딩**
```text
docs/기능명세서.md의 F-01, F-02와 목업의 온보딩 화면(s0~s6)을 읽어줘.
아바타 빌더 스텝과 꾹꾹이 로더를 포함한 온보딩 플로우를 만들고,
storage 어댑터에 저장되게 해줘. 진입 화면 카피와 스텝 구성은 목업 그대로.
```

**M2 — 홈 + 카드 매칭**
```text
F-03의 홈 화면과 F-04의 카드 매칭 엔진을 만들어줘. 매칭 점수 규칙은
명세대로: 생애단계 일치 3점 / 품종그룹 일치 3점 / 조건태그 개당 2점 /
최신성 1점. cards.json을 점수순으로 렌더링하고, 읽은 카드는 점수 -50%.
```

**M3 — 증상 로그 + 오늘의 체크**
```text
F-05 증상 기록(3탭 이내 완료 규칙, 레드플래그 태그 선택 시 즉시 병원
안내 모달)과 F-15 오늘의 체크 배너(답변 → 습관 프로필 저장 + 그 자리에서
매칭 카드 노출)를 만들어줘. 동작은 목업의 오늘의 체크와 동일하게.
```

**M4 — 마감**
```text
F-07 체크리스트(완료 시 발도장 팝 애니메이션), 목업의 감성 모듈들
(시간대별 인사말, 생애단계별 말풍선 + 귀 쫑긋, 계절 배너, 오늘의 냥지식),
하단 고정 면책 문구까지 추가해줘. 마지막으로 목업과 나란히 비교하면서
빠진 동작이 없는지 QA해줘.
```

## 팁
- 한 세션 = 한 마일스톤. 끝날 때마다 git commit + Vercel 확인 후 다음 세션.
- 막히면 "docs/찐집사_목업.html이랑 비교해서 동작을 맞춰줘" 한 줄이 제일 잘 먹힘.
- M0의 /dev/avatar 페이지에서 144조합을 눈으로 확인하고, 이상한 조합은 그 자리에서 고치기.
