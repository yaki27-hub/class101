## 2. 포지셔닝

> "우리 애 이거 먹어도 돼요?"부터 "얘 왜 이래요?"까지 —
> **내 고양이를 기억하는, 편하게 물어보는 건강 챗봇.**

범용 AI와의 차이 = **기억**. 프로필 + 습관 베이스라인 + 증상 히스토리를 대조해 "평소랑 다른지"를 아는 것이 유일한 차별점이며, 따라서 **기록 루프(F-05, F-15)는 챗봇의 부속이 아니라 챗봇을 성립시키는 전제 조건**으로 MVP에 포함한다.

## 4. 유저 플로우 (MVP)

```
스플래시 (브랜드 카피)
 → 카카오 로그인 (F-13')
 → 묘 프로필 6스텝 (F-01, 아바타 빌더 포함)
 → 첫 결과: 생애 시계 + "○○에 대해 궁금한 걸 물어보세요" (F-02)
 → 챗봇 홈 (F-03')
     ├─ 대화 (F-08) ── 레드플래그? ─→ 즉시 병원 안내 + 24시 병원 링크 (F-09)
     │      └─ 대화 종료 → "증상 기록으로 남길까요?" 원탭 (F-05)
     ├─ 오늘의 체크 1문항 (F-15) → 습관 프로필 축적
     └─ 추천 질문 칩: 프로필 기반 생성 ("7살부터 챙길 검진은?" 등) — 첫 대화 진입장벽 제거
```

**추천 질문 칩이 중요한 이유**: 챗봇 앱 최대 이탈 지점은 빈 입력창 앞의 막막함. 프로필(나이·품종·조건)로 개인화된 질문 3개를 항상 노출해 첫 대화 성공률을 끌어올린다.

## 5. 챗봇 상세 (F-08 보강)

기본 설계는 docs/챗봇_시스템프롬프트.md 그대로 (5블록 포맷, NUMBER RULE, R1~R6 안전 규칙, 히스토리 대조). 이번 피벗에서 추가되는 것:

**5-1. 사료/간식 질문 답변 정책 (신규 — R7로 프롬프트에 추가)**
- "이 사료 어때요?" → 특정 제품의 좋고 나쁨을 단정하지 않는다. 대신 **이 고양이 기준의 선택 체크리스트**로 답한다: 생애 단계 적합성(키튼/성묘/시니어 표기), 주요 성분에서 볼 것, 급여량 계산(체중 기반), 전환 시 7일 규칙
- "간식 너무 짜지 않아요?" → 간식은 하루 권장 열량의 10% 이내 원칙 + 이 고양이 체중 기준 환산치 제시
- 처방식(신장·요로 등)은 반드시 수의사 처방 영역임을 명시
- 제품 비교·순위 요청 → "많이 먹는 것 ≠ 좋은 것" 원칙 안내, v1.1 유사묘 통계 출시 전까지 데이터 기반 답변 유보

**5-2. 대화 → 기록 전환 (챗봇 기억의 공급로)**
- 증상성 대화(구토·설사·식욕 등 태그 감지) 종료 시: "오늘 대화를 츄르의 증상 기록으로 남길까요?" → 원탭 시 태그·요약 자동 생성해 symptom_logs 저장
- 이 전환율이 서비스의 생명선 지표 (§8)

**5-3. 대화 UX**
- 스트리밍 응답, 5블록 헤더는 접이식 카드로 렌더링 (🚦 판정 블록은 항상 펼침 + 색상 강조)
- 사진 첨부 지원 (토사물·피부·검진지)
- 이전 대화 이어보기 (세션 리스트)

## 7. 구현 방식 결정 — 모바일 앱

**권장: 1단계 웹앱(PWA) → 2단계 스토어 앱 (Expo 래핑)**

| | 1단계: Next.js 웹앱 + PWA | 2단계: React Native/Expo |
|---|---|---|
| 장점 | 익숙한 스택, 심사 없음, 커뮤니티에 **링크 한 줄로 배포** → 유저 테스트에 최적, 수정 즉시 반영 | 푸시 알림 강력(오늘의 체크 리마인드), 스토어 노출, 네이티브 UX |
| 단점 | iOS 푸시 제약, 스토어 노출 없음 | 심사(건강 정보 앱은 의료 조언 경계·면책 확인), 배포 사이클 느림 |

유저 테스트 단계에서는 링크 배포가 압도적으로 유리하므로 **PWA로 검증 → PMF 신호 확인 후 Expo 전환**. 러닝앱 때 RN/Expo 경험이 있어 2단계 전환 리스크는 낮음. 스토어 심사 대비로 면책 문구·"의료 조언 아님" 고지는 1단계부터 정식 탑재.

## 8. 유저 테스트 계획 (애묘인 커뮤니티)

**검증 가설 3개**
1. (신뢰) 집사들은 출처가 표기된 챗봇 답변을 출처 없는 답변보다 신뢰하는가 — A/B 또는 인터뷰 비교
2. (기억) "평소보다 잦아요 / 평소 패턴 안이에요" 응답을 받았을 때, 기록을 계속할 동기가 생기는가
3. (습관) 고양이가 아플 때 검색·카페 대신 이 앱을 먼저 여는가 (2주 추적)

**정량 지표 (테스트 기간 2~4주)**
- 가입 → 첫 대화 완료율 ≥ 60% (추천 질문 칩 효과 포함)
- 대화 → 증상 기록 전환율 ≥ 30% (§5-2, 생명선 지표)
- 7일 재방문율 ≥ 30% / 오늘의 체크 응답률 ≥ 40%
- 레드플래그 오탐·미탐 사례 전수 수집 (안전 QA)

**모집**: 네이버 고양이 카페(치치카 때 채널 재활용), 인스타 애묘 계정, 지인 집사 10~20명 클로즈드 베타 → 커뮤니티 오픈 베타. 인터뷰는 5~8명 심층(30분, 실사용 화면 녹화 리뷰).
