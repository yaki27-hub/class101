- 참고: 킥오프킷의 "MVP에 Supabase·로그인·챗봇 없음" 스코프는 v1 기준 → **기능명세서 v2가 우선**

- 영향받는 태스크: T-01(Next.js 스캐폴드로 확정), T-21(배포 채널 = 웹 링크, 미정-7 부분 해소)

## D-03 (2026-07-19) 디자인 시스템 → **Clay 스타일 참조 (`docs/디자인시스템_Clay.md`)**

## D-09 (2026-07-20) T-16 동기화 인증 → **Supabase 익명 로그인**

- 선택: Supabase Anonymous 로그인으로 기기별 auth.uid() 확보 → RLS 통과, 카카오 없이 동기화. 추후 카카오 로그인 시 익명 계정 연결(link)
- 제약: ① 이 샌드박스는 Supabase 접속 차단 → 동기화 코드는 배포본에서만 검증 ② 사진/일러스트 컬럼이 스키마에 없어 마이그레이션 0002 필요
- 안전 원칙: SupabaseStorageAdapter 교체는 NEXT_PUBLIC_USE_SUPABASE=1 플래그 뒤에 두어, 검증 전까지 기본 localStorage 유지(현재 동작 보호)
- 선행 조건(사용자): (a) 0002 마이그레이션 SQL 적용 (b) Supabase 대시보드 Auth→Anonymous 로그인 활성화
- 영향: T-16 어댑터는 위 선행 완료 후 구현·배포·검증
