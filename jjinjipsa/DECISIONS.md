## D-00 (2026-07-19) 킥오프 전 이미 확정된 제약 (1-Pager + 기능명세서 v2)

- 고양이 전용. 개·타 반려동물 확장 안 함 → 안전 룰 단일화, 고양이 특화가 해자
- 챗봇이 MVP 코어 (v1 카드 피드 중심에서 피벗). 기록 루프(F-05, F-15)는 챗봇의 전제 조건으로 MVP 포함
- 카카오 로그인 MVP 필수 (비용 통제 + 히스토리 동기화). 비로그인 체험 모드 없음
- 당분간 무료. 과금 설계 없음, 비용 통제 장치만 탑재 (일 10회 한도·모델 라우팅·룰 선차단)
- 백엔드 Supabase (Postgres + Auth)
- PG 직접 연동 금지 — 추후 판매는 Groble(groble.im) 링크 자리만
- API 키 없는 구간은 mock 어댑터 + 교체 지점 파일 분리
- 참고: 킥오프킷의 "MVP에 Supabase·로그인·챗봇 없음" 스코프는 v1 기준 → **기능명세서 v2가 우선**

## D-01 (2026-07-19) 플랫폼: Expo 즉시 vs PWA 먼저 → **PWA 먼저 (Next.js)**

- 선택: Next.js 웹앱 + PWA로 MVP 구축, PMF 신호 확인 후 Expo 전환 (명세서 v2 §7 권장안)
- 이유: 커뮤니티 유저 테스트에 링크 배포가 압도적으로 유리, 심사 없음, 수정 즉시 반영. 러닝앱 RN/Expo 경험이 있어 2단계 전환 리스크 낮음
- 후속 조치: 스토어 심사 대비 면책 문구·"의료 조언 아님" 고지는 1단계부터 정식 탑재. 저장·LLM 어댑터는 Expo 전환 시 재사용 가능하게 UI와 분리
- 영향받는 태스크: T-01(Next.js 스캐폴드로 확정), T-21(배포 채널 = 웹 링크, 미정-7 부분 해소)
