## 1. 제품 개요

- **한 줄**: "우리 애 이거 먹어도 돼요?"부터 "얘 왜 이래요?"까지 — **내 고양이를 기억하는 건강 챗봇**
- **타겟**: 한국 고양이 집사 (고양이 전용, 개 확장 안 함 — 안전 룰 이원화 방지 + 고양이 특화가 해자)
- **핵심 가치 (재방문 이유)**: 범용 AI와의 차이 = **기억**.
  프로필 + 습관 베이스라인(오늘의 체크) + 증상 히스토리를 대조해 "평소랑 다른지"를 아는 것.
  → 따라서 기록 루프(F-05, F-15)는 부속이 아니라 **챗봇을 성립시키는 전제 조건**.
- **원칙**: 출처 있는 정보 · 광고 없는 중립성 · 안전 최우선(레드플래그 룰이 AI를 우회) · 의료 조언 아님 고지 상시.

## 2. MVP 범위

### 넣을 것 (기능명세서 v2 §3)
| 우선순위 | 기능 |
|---|---|
| P0 | 카카오 로그인(F-13'), 묘 프로필 6스텝+아바타(F-01), 생애 시계 첫 결과(F-02), **컨텍스트 챗봇(F-08)**, 레드플래그 룰 엔진(F-09), 대화→증상 기록 원탭(F-05), 비용 통제(COST) |
| P1 | 오늘의 체크 하루 1문항(F-15), 챗봇 중심 홈(F-03'), 케어 카드 16장=챗봇 지식 베이스(F-04'), 생애 단계 체크리스트(F-07) |

### 일부러 안 넣을 것 (v1.1 이후)
- 건강 이벤트 상세(F-06), 공유 카드(F-10), 학술 파이프라인(F-11), 병원 찾기(F-12 — 단 레드플래그 시 24시 병원 지도 **링크아웃**만 최소 탑재), 유사묘 통계(F-14), 입양 연결(F-16)
- **과금 전체** (당분간 무료, 비용 통제 장치만 탑재)
- PG 직접 연동 — 절대 금지. 추후 판매는 Groble(groble.im) 링크를 붙일 자리만.

## 4. 데이터 모델 초안 (Supabase Postgres)

> 상세 필드는 태스크 착수 시 확정. RLS: 기본 = 본인 소유 행만. 가족 공유는 [미정-5].

- `profiles` — id(auth.users FK), nickname, created_at
- `cats` — id, owner_id, name, birth_date, birth_estimated, gender, neutered, breed_group, weight_kg, conditions[], avatar_fur, avatar_pattern, avatar_eyes
- `chat_sessions` — id, cat_id, user_id, title, started_at
- `chat_messages` — id, session_id, role, content, image_url, model, token_in/out
- `symptom_logs` — id, cat_id, tags[], summary, source(chat|manual), chat_session_id, occurred_at
- `trait_answers` — id, cat_id, question_key, answer, answered_on  ← 오늘의 체크(F-15) 습관 베이스라인
- `usage_counters` — user_id, date, chat_count  ← 일일 10회 한도
- `care_cards` — 정적 시드 16장 (title, body, source_name, tags_stage, tags_breed, tags_condition, severity)
- 레드플래그 룰 — DB 아닌 **코드 내 룰 테이블** (버전 관리 + AI 호출 전 선차단)

## 5. 마일스톤

### M1 — 핵심 가설 검증 (최소 화면)
> **가설: "내 고양이 프로필을 기억하는 챗봇 답변은 범용 AI보다 낫다"** — 프로필을 바꾸면 추천 질문 칩과 답변이 달라지는 것을 눈으로 확인
- 앱 셸 + 디자인 토큰 → 최소 프로필 입력 → 생애 시계 → 챗봇 대화(LLM 어댑터, mock 폴백) + 프로필 컨텍스트 주입 + 추천 질문 칩 3개
- 로그인·Supabase 없이 로컬 저장으로 시작 (저장 어댑터 뒤에 숨겨 교체 가능하게)

### M2 — 안전 + 기억 루프
- 레드플래그 룰 엔진(AI 선차단 + 병원 안내 + 24시 병원 링크아웃)
- 대화→증상 기록 원탭 전환 (생명선 지표), 수동 기록(보조)
- 오늘의 체크 1문항 → 습관 베이스라인 → 답변에 "평소 패턴" 대조 반영

### M3 — 출시 준비
- 카카오 로그인 + Supabase 스키마·RLS + 로컬→서버 동기화
- 비용 통제: 일일 한도, 모델 라우팅, 사용량 카운터
- 아바타 빌더 6스텝 완성, 케어 카드 16장 연결("관련 카드 보기"), 면책 문구, 유저 테스트 배포 준비 (§8 지표 계측 포함)
